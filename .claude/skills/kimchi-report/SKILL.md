# kimchi-report — 보고서 1장 작성

## 사전 조건
- `study/`에 논문 스터디 노트가 있어야 함. 특허도 정해졌으면 더 좋다(없으면 먼저 `kimchi-find`로 특허를 찾자고 안내).

## 보고서 구조 (CLAUDE.md 규약)
제목: **"김치, 전통식품을 넘어 — ○○ 기술의 현재"**
1. **여는 한 문단** — 왜 이 주제인가(문제 제기, 일상 언어).
2. **[논문] 연구는 여기까지 왔다** — 무엇을·어떻게·결과. **핵심 숫자 1개** 굵게. 출처(학술지·권호·연도) 명시.
3. **[특허] 산업은 이렇게 응용한다** — 어떤 문제를 어떻게 해결하는 기술인가. **핵심 숫자/효과 1개**. 출처(특허번호·출원인·연도).
4. **한 줄 결론** — 연구와 산업을 잇는 메시지.
5. **출처** — 논문·특허 정식 서지.

## 절차
1. 노트와 특허 정보를 모아 위 구조로 `report/report.md`를 쓴다(분량 = A4 1장, 약 600~900자 본문).
2. 과장·추측 금지. 숫자·인용은 출처 있는 것만.
3. 초안 후 사용자에게 "고치고 싶은 부분?"을 묻고 반영한다.
4. 확정되면 원하면 한글 워드/PDF 변환 방법을 안내(pandoc 또는 복붙).

## 규칙 (저작권·정확성)
- 본문 복사 금지, 자기 말 요약. 직접 인용 15단어 미만·1회·출처 포함.
- 출처 못 찾은 문장은 뺀다. 최종 사실 확인은 원문 기준.
- 모든 출력은 한국어, 간결·정확.
