# CLAUDE.md — 김치 논문 보고서 프로젝트 지침

이 파일은 Claude Code가 이 폴더에서 일할 때 따르는 규칙입니다.

## 프로젝트 목적
김치 전문가 과정 과제: **사이언스온(ScienceON)에서 김치 관련 논문/특허를 찾아 → 읽고 → 보고서 1장을 작성**한다. 제출 기한 **6월 18일**.

## 작업자
- 김치 전문가 과정 수강생(비개발자).
- 코드·터미널에 익숙하지 않음. **쉬운 한국어로, 한 번에 한 단계씩** 안내할 것.
- 전문 용어는 풀어서 설명하고, 다음에 뭘 하면 되는지 항상 한 줄로 알려줄 것.

## 폴더 구조
- `topics.md` — 8개 추천 주제 + 박성희 박사 트랙 + 검색 키워드 (주제 고를 때 참고)
- `papers/` — 사용자가 내려받은 논문 PDF를 넣는 곳
- `study/` — 논문별 스터디 노트(요약·핵심)를 저장하는 곳
- `report/report.md` — 최종 보고서 1장

## 워크플로우 (4단계)
1. **주제 고르기** — `topics.md`를 함께 보고, 차별화·관심에 맞는 주제 1개를 정한다.
2. **논문 찾기·다운로드** — 사이언스온은 로그인·다운로드가 필요할 수 있다. Claude가 직접 받지 못하면, 검색 키워드와 찾는 법을 안내하고 사용자가 PDF를 `papers/`에 저장하게 한다. 웹 검색으로 후보 논문·특허를 추천하는 건 적극적으로 돕는다.
3. **스터디** — `papers/`의 PDF를 읽고, 쉬운 말로 요약 + 핵심 숫자·결론·한계를 정리해 `study/파일명.md`로 저장한다.
4. **보고서 작성** — 아래 구조로 `report/report.md`를 작성한다.

## 보고서 1장 구조 (권장)
논문 1편 + 특허 1건을 **짝지어** 비교하면 1장에 스토리가 생긴다.
- 제목: "김치, 전통식품을 넘어 — ○○ 기술의 현재"
- 상단: **논문** 1편 — "연구는 여기까지 왔다"
- 하단: **특허** 1건 — "산업은 이렇게 응용한다"
- 각 항목에 **핵심 숫자 1개**(예: 흡착률 87%, 반감기 5배, 수출 1.5억 달러)
- 한 줄 결론 + **출처(학술지명·특허번호·연도) 반드시 명시**

## 저작권·인용 규칙 (중요)
- 논문 본문을 **그대로 복사하지 않는다.** 항상 **자기 말로 요약**한다.
- 꼭 필요한 직접 인용은 **15단어 미만, 한 출처당 1회**, 따옴표와 출처를 함께 단다.
- 그림·표는 캡션으로 출처를 밝히고, 보고서엔 직접 다시 그리거나 요약 수치만 쓴다.
- 확실하지 않은 사실·인용은 넣지 않는다. 출처를 못 찾으면 그 문장을 뺀다.

## 출력 규칙
- 모든 답변·문서는 **한국어**.
- 보고서는 과제 제출용이므로 **간결하고 정확하게**. 과장·추측 금지.
- 사실 확인이 필요한 최신 정보는 웹 검색으로 확인하고 출처를 남긴다.

## 한 가지 더
배지·점수·추천은 참고일 뿐, **최종 사실 확인은 사이언스온 원문**으로 한다. 이게 과제의 핵심이기도 하다.
